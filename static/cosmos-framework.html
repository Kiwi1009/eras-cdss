GPU-Accelerated Multi-Agent Large Language Models for Enhanced Recovery After Surgery Protocols

Multi-Agent Dialogue System with RAG for Enhanced Recovery After Surgery (ERAS) in Thoracic Procedures

Hsing-Hsien Wu, I-Ni Wu, Ya-Chuan Chen, Hsiang-Wei Hu, Dawn Chen
Division of Thoracic Surgery, Tainan Municipal Hospital, Show Chwan Healthcare System, Taiwan;
Department of Artificial Intelligence in Healthcare, International Academia of Biomedical Innovation Technology, Reno, Nevada, United States

Abstract

Background: Enhanced Recovery After Surgery (ERAS) protocols require real-time, personalized clinical decision support, but traditional CPU-based rule engines suffer from high latency and low throughput, limiting multi-patient concurrent management capabilities.

Objective: This study evaluates the performance gains of GPU-accelerated Large Language Model (LLM) inference systems in ERAS clinical decision-making and validates their feasibility in real-world clinical scenarios.

Methods: We developed a multi-agent LLM system powered by NVIDIA RTX 5090 GPU, integrating a hybrid Retrieval-Augmented Generation (RAG) architecture, compared against traditional CPU rule engines. The system addresses three core ERAS use cases: Postoperative Nausea and Vomiting (PONV) prevention, Postoperative Delirium (POD) checklists, and chest tube removal decisions. Through microbenchmarks, stress testing, and clinical task accuracy evaluation, we quantified system performance and clinical value.

Results: The GPU-accelerated system achieved significant performance improvements over the CPU baseline: 2.9× reduction in mean latency (650 ms vs 1900 ms), 2.4× reduction in P95 latency (850 ms vs 2050 ms), and 7× increase in maximum throughput (3.5 QPS vs 0.5 QPS). Under 16-patient concurrent stress testing, P95 latency remained under 2.1 seconds with GPU utilization at 60–80% and memory usage at 16–18 GB (70–75% VRAM). Clinical task accuracy reached 95%, demonstrating flexible edge-case handling.

Conclusion: A single RTX 5090 GPU enables bedside real-time response (P95 <1 s) while supporting 8–16 concurrent patient management, providing a viable solution for resource-constrained on-premise deployments. The open-source software stack and clear cost-benefit analysis (4–10 month ROI) demonstrate clinical deployment potential. Future directions include TensorRT-LLM quantization, cuVS GPU vector retrieval acceleration, multi-GPU scaling, and multimodal RAG integration.

System Architecture: Five-Layer Modular Design

The system integrates EHR data via FHIR R4, retrieves clinical knowledge through hybrid RAG (BM25 + semantic search), processes queries using GPU-accelerated LLM inference (RTX 5090, GPT-OSS-20B), coordinates multi-agent reasoning (Surgeon, Anesthesia, Nurse), and outputs real-time ERAS recommendations.

Data Layer FHIR-compliant EHR integration | ERAS guidelines indexing | HIPAA/GDPR synthetic data
RAG Layer BM25 keyword + semantic embedding retrieval | ChromaDB (CPU) → cuVS (GPU roadmap)
Inference Layer GPT-OSS-20B (FP16, 8K context) | Ollama server | RTX 5090 (24 GB, 21,760 CUDA cores)
Multi-Agent Layer 3 specialized agents → debate → arbitrator → consensus decision
Output Layer PONV prevention | POD checklist | Chest tube decisions (all <1s latency)

Discussion

On a single on-prem RTX 5090, our ERAS assistant achieves bedside-ready performance. Mean latency fell from ~1900 to ~650 ms and P95 from ~2050 to ~850 ms; throughput reached ~3.5 QPS, supporting 8–16 patients concurrently. Across 30 ERAS prompts (PONV, POD, chest-tube), keyword-based accuracy improved from 84% to 95% with traceable rationales. Stress tests showed near-linear scaling to 8-way concurrency and stable operation (VRAM 70–85%, 65–75 °C). These results demonstrate reliable, auditable real-time support.
